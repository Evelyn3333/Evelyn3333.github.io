如何用-GitHub-快速创建自己的专属博客(三)

04-05-2018
本文总阅读量次

阅读本文可能需要10分钟…
hexo 的安装指北

首先确保你的电脑上已经安装了 git 和 node.js. 在 terminal(终端)输入,

git version

如果你看到了如下结果, 那么说明安装成功, 否则左转传送门Git 安装

查看是否已经安装了 node.js. 在 terminal 输入,

npm -v

如果你看到了版本号码, 说明安装成功, 否则继续左转传送门Node.js 下载
注: windows 和 mac 的指令可能有所不同, 对于版本验证的指令可以尝试(-v, -version, 和version)

以上步骤完成之后就可以开始安装 hexo 了!~!~

继续留在终端, 指定一个你想要保存 hexo 文件的路径, 创建一个文件夹, 使用如下指令

mkdir ~/你自定义的路径/文件夹的名字

到你刚刚创建好的路径下面,

cd ~/你自定义的路径/文件夹的名字

输入指令,

npm -install hexo --save

耐心等待安装完成, 输入,

hexo -v

如果出现了类似的信息说明安装成功,

接下来初始化 hexo, 继续在终端输入,

hexo init

接下来安装组件, 输入

npm install

耐心等待安装完成, 现在你可以尝试 generate 你的本地页面了, 输入,

hexo g

然后启用本地服务器, 

hexo s

不要关闭这个 terminal 创空, 在你的浏览器中输入http://localhost:4000/
或者可以直接复制终端提示的地址, 你就可以看到本地生成的一个基础的 blog 页面啦~

你已经成功的安装了 hexo 在本地.

接下来只需要简单的设置就可以把本地生成的 blog 和你的远程服务器关联起来了.
Ctrl + c 终止本地 server,这样你就可以直接跳回 hexo 的文件夹, 或者另外开启一个 terminal, 使用 cd 命令重新进入到 hexo 的目录下.

首先配置你的用户信息,

在终端输入,

git config --global user.name "yourUserName"
git config --global user.email "yourEmail"

–global 表示设置为全局变量.

确保当前位置是你的 hexo 目录, 输入, 

vim _config.yml

注意下划线, 如果你打开的是一个空白文档, 说明你的目录不正确, 使用Esc + :q 的命令退出界面, 重新确认当前路径是否为 hexo 的安装目录.
在文件中找到 deploy, 更改为如下内容, example 改为你自己的用户名.
使用i 切换到编辑模式, 更改之后使用 Esc + :wq 退出并保存.

退出 vim 界面之后, 继续输入, 

hexo d

terminal 会提示你输入用户名和密码(github 的用户名和密码), 按照提示输入就可以了, 这里面输入密码的时候不会有显示, 自己确保输入正确就可以了.

这样你在本地生成的静态页面就成功部署到 github 上面了, 现在你可以使用定义好的域名来访问你的博客啦~~
