如何用-GitHub-快速创建自己的专属博客(四)

04-05-2018
本文总阅读量次

阅读本文需要10分钟.
熟练掌握可能要更久….

现在你已经有了一个基本款的 hexo 生成的博客了, 那么你就可以选择, 练习前端技能(自己写一个静态页面), 拆轮子(拿现成的主题来用), 以及直接开始写博客了.

这里简要描述一下下载和部署已有的主题, 以及简单的 hexo 命令.

首先你可以到 hexo 官网找一个心仪的主题, 点击主题的名字(不是预览图片), 然后页面会自动跳转到开发者的 github 页面.

通常 README 中会有非常详细的部署细节. 主要步骤就是 安装, 配置, 和 deploy.

安装主题

  • 在页面中复制主题地址, 选择 clone or download 在下拉菜单中复制.

  • 在 terminal 中, cd 进入到 hexo 的安装目录, 输入,

    $ git clone http://github.com/hexojs/这一段都是复制的内容.git themes/主题名字`

这里需要注意的是, 通常文档中已经给了命令行, 所以就直接复制就可以了, 不需要这么麻烦. themes/name 的参数是为了定位到 themes 的文件夹下面创建一个和主题名字一致的文档.

配置文件

  • 执行成功之后在 terminal 中继续输入,配置文件.

    vim _config.yml

这时候你可以修改 blog 里面的一些个人信息例如, author, title, description 等等.
最重要的是, 在 theme 字段中, 把主题更改为你刚刚下载的那个主题的名字. 

theme:  你下载的主题

更改之后Esc + :wq 退出. 这里仍然需要提醒的是, 如果你打开的文本为空,那么退出重新确认路径.

部署页面

  • 现在执行如下命令, 清除本地缓存, 之后每一次的 deploy 都建议清除一下本地的缓存, 否则可能会出现无法更新的问题.

    hexo clean

  • 接着输入, 编译你的本地页面,

    hexo g

  • 部署页面,

    hexo d

  • 现在就可以通过域名访问你的主页啦~

接下来是一些简单的 hexo 命令, 这些命令都是在 hexo 的安装目录下通过 terminal 来执行.

  1. 新建一个文章

    这里需要注意的是, 如果你创建的名字有空格, 那么就需要用双引号括起来. 新建的文件保存在hexo 安装目录下的 source/_posts 里面. 所以你也可以通过访问这个文件夹直接创建一个 md 格式的文件. markdown 的语法参见Google/Baidu.

  2. 编译(Generate)

    通常编译之前建议先使用

编译之后本地会自动生成一个 public 的文件夹, 如果不清空的话, 本地文件在部署的时候有可能会出现无法更新等问题. 使用 generate 命令之后, 所有 md 文件会被编译成 html 的文件格式. 详情参考Wikipedia.

  1. 开启本地服务器

本地服务器开启之后会自动监听4000端口, 通过浏览器访问 localhost:4000 可以在本地看到刚刚编译成功的静态页面, 同时, 在更新或者编辑 md 文件时, 不需要重新编译, 只需要保存之后刷新本地页面就可以看到更改后的内容.

  1. 部署(deploy)

    将本地编译后的页面部署到 git 上面.
