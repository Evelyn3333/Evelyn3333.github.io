如何用-GitHub-快速创建自己的专属博客(二)

04-03-2018
本文总阅读量次 
Github 个人主页创建成功之后, 域名绑定, 以及域名解析失效问题的解决办法.
  • 阅读本文需要2分钟

注册域名通常会去Godaddy, 或者DNSPod.
注册好域名之后只需要去你的个人账户里面找到 DNS 的选项,

添加一个 CNAME 的记录, 类型选择 CNAME, 名称选择 www, 指向的是你个人主页的地址, TTL 可改可不改, godaddy 默认的最小值是600s.

这里需要一提的是, Apex(A 类型)的域名解析时间非常的不确定, 10分钟到一整天时间不等. 所以其实完全没有必要冒着域名解析失败网站挂掉的危险去设置. 设定 Apex 域名的主要原因是使别人可以不加 www 也可以访问到你.

如果你一定要设定的话…
在 terminal 输入

example 替换成你自己的用户名; 或者使用命令

注: 后面的两条参数是为了过滤掉多余的信息, 可以不加.
你就会看到这个网站的 IP 地址, 在刚刚修改 DNS 的地方加入一条 类型为 A 的信息, 指向( Value) 设置成 IP 地址. 其余设定相同.
你的网站有可能会有两条不同的地址, 你需要把两条地址都加入到 DNS 中去.

我的建议是不!要!加!

如果你一定要加的话…
在 DNS 中添加这两条 IP 地址 (美国地区测试有效), 设定方法同上

绑定域名的方法主要有三种:

  1. 直接在 Repository 的 setting 中更改并保存.

    保存之后你的库中就会自动生成一个 CNAME 的文件, 文件只有一行内容, 就是你的自定义域名.可以看到这里面可以看到, 最后一栏里面如果你使用了自定义的域名, 那么默认 https 的协议将无法被应用.

  2. 远程生成一个 CNAME 的文件, 里面添加你的域名地址.

  3. 使用 hexo 的话, 上面两条都会失效, 因为每一次 deploy 的时候这个文件就会消失了, 你需要在本地的 source 文件夹里面创建一个 CNAME 的文件, 同样的, 需要写上你的域名地址.
